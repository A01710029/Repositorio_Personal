¿Qué diferencias y semejanzas hay entre Java y JavaScript?

Java y Javascript, aunque ambos son lenguajes de programación comúnmente utilizados en la industria que comparten la sintaxis básica, como sus nombres similares, difieren en muchas de sus características. Para empezar, Java es un lenguaje de programación de propósito general que tiene una variedad de usos, mientras que JavaScript es un lenguaje de programación interpretado y principalmente se utiliza en el desarrollo de páginas web. Relacionado a esto, programas de Java deben ser ejecutados dentro de la Máquina Virtual Java, mientras que los programas de JavaScript pueden ser ejecutados dentro de un navegador web o Node.js. Por otro lado, tienen diferencias notables en sus estructuras internas. Java usa un tipado estático, requiriendo que las variables sean declaradas explícitamente, mientras que JavaScript usa un tipado dinámico, lo que significa que el tipo de las variables puede cambiar en cualquier parte del programa. Además, Java sigue el paradigma de orientación a objetos para herencia, mientras que JavaScript utiliza prototipos en vez de clases en su proceso de herencia de objetos.

¿Qué métodos tiene el objeto Date? (Menciona al menos 5)

El objeto Date tiene los métodos getDate(), que permite regresar el día del mes para una fecha específica, getMonth(), que permite regresar el mes para una fecha específica, getFullYear(), que permite regresar el año para una fecha específica, getHours(), que permite regresar la hora para una fecha específica según la hora local y getTime(), que permite regresar la fecha y hora específicas en milisegundos según la hora universal.

¿Qué métodos tienen los arreglos? (Menciona al menos 5)

Los arreglos tienen los métodos push(), que permite agregar elementos al final de un arreglo, pop(), que permite eliminar y regresar el último elemento de un arreglo, splice(), que permite eliminar un elemento en un índice específico en un arreglo, indexOf(), que permite regresar el índice de la primera instancia de un elemento específico en un arreglo (o -1 si no existe) y forEach(), que permite ejecutar una función de manera iterativa a cada elemento de un arreglo, simulando un for loop.

¿Cómo se declara una variable con alcance local dentro de una función?

Para declarar una variable con alcance local dentro de una función, se debe utilizar las claves var, let o const en su declaración, antes del nombre de la variable.

¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

Utilizar variables globales dentro de funciones implica que estas variables pueden ser accedidas y modificadas en cualquier parte del código por cualquier función. Esta es una mala práctica, ya que puede complicar el debugging, porque no se puede rastrear fácilmente las modificaciones de la variable. Esto en sí también afecta la predictibilidad y reutilización de las funciones dependientes de estas variables, ya que las variables globales pueden ser modificadas fuera del contexto de su uso dentro de la misma. Por otro lado, pone en riesgo la seguridad del código, por su condición de ser modificables en cualquier y por cualquier parte del código.
