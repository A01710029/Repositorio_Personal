Navegación

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Es buena práctica usar JavaScript para checar la validez de los inputs de las formas antes de enviar los datos al servidor ya que nos permite validar desde el lado del usuario, lo que tiene muchos beneficios. Para empezar, permite reconocer errores en los inputs antes de mandar una forma, lo que reduce el tiempo de procesamiento, el uso de ancho de banda y la carga en el servidor, al solo mandar la forma cuando todo está validado. También, se considera una práctica importante de seguridad al evitar que datos no válidos lleguen al servidor, ya que no se mandan hasta después de cumplir ciertas reglas.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Se puede saltar la seguridad de validaciones hechas con JavaScript con una serie de modificaciones, ya que las validaciones ocurren desde el lado del usuario. Para empezar, lo más fácil es desactivar JavaScript en el navegador, una opción presente en la configuración del mismo. Ligeramente más complicado, se puede abrir las herramientas de desarrollo del navegador y modificar el código desde ahí o utilizar bots para sobrepasar la forma por completo y mandar los datos directamente al servidor.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Aunque se puede saltar la seguridad de las validaciones de JavaScript, aún así se considera buena práctica utilizarlas porque es una de las mejores maneras existentes de validar las respuestas de manera rápida, eficiente y con la menor carga y comunicación de datos errónea entre el navegador y el servidor. Puede ser que no sea una buena medida de seguridad, pero este aspecto se puede complementar con otras medidas menos fáciles de sobrepasar y no elimina los beneficios de velocidad y retroalimentación directa al usuario.
